## 一、产品定位

### 使用者
本系统的主要使用者是**高校辅导员**，而非学生本人。辅导员通过系统录入学生信息或上传简历，获得 AI 分析结果后，用于：

- 对学生进行成长路径分析、短板诊断
- 制定个性化帮扶方案
- 准备谈心谈话（切入话题、关注重点、风险预警）
- 在保研、升学、就业指导中进行证据化支持
- 生成可存档的 Word 正式评估报告

### 设计原则
- **辅助而非替代**：所有评估结果仅供辅导员参考，最终决策由辅导员结合校政策、学生意愿、导师意见综合作出。
- **数据安全优先**：相似案例匿名化、本地化存储、可完全离线降级（无 API Key 时走启发式规则）。
- **可解释性**：每一个评分、每一条推荐都附明确的依据说明，避免"黑盒"。
- **成本可控**：缓存优先、小模型兜底、Top-K 截断、规则降级，LLM 调用可控在最小必要范围。

## 二、功能概览

### 2.1 用户端（辅导员工作台）

**录入与识别**
- 三种录入方式：学生自述文本、上传简历（PDF / DOCX / TXT / MD）、快速表单
- 大模型画像抽取：从非结构化文本中识别专业、绩点、英语成绩、竞赛、论文、实习、学生工作、目标意向等
- 启发式兜底：未配置 API Key 时用正则/关键词规则抽取，系统仍可正常使用
- 画像核对：抽取完成后提供可编辑界面，辅导员可微调绩点、英语、竞赛等字段后再生成推荐

**六维能力画像**（0-100 分）

| 维度 | Key | 合成逻辑 |
|------|-----|----------|
| 学业基础 | academic_foundation | 0.70 × 绩点分 + 0.20 × 英语分 + 0.10 × 荣誉分 |
| 科研创新 | research_innovation | 0.65 × 论文分 + 0.35 × 科研经历分 |
| 竞赛实践 | competition_practice | 竞赛综合分（Top3 加权 0.6/0.3/0.1） |
| 工程实践 | engineering_practice | 实习经历分（含项目/技能信号） |
| 组织服务 | organization_service | 0.70 × 学生工作分 + 0.30 × 志愿服务分 |
| 成长规划 | growth_planning | 启发式：目标关键词 + 经历广度 + 跨域积累 + 无明显短板 |

**九条成长路径评估**
1. 普通推免
2. 特殊专长 A 类（竞赛/科研方向）
3. 特殊专长 B / 本硕博贯通
4. 特殊专长 C / 辅导员计划
5. 支教计划
6. 工程专项 / 专硕
7. 实习就业
8. 考研（最低优先级兜底路径）
9. 考公（最低优先级兜底路径）

**分析与推荐**
- 规则评分：对 9 条路径分别设置差异化权重公式
- TOPSIS 多指标综合评价：逼近理想解排序，输出算法评分
- 评分融合：`final_score = 0.75 × rule_score + 0.25 × algorithm_score`
- 意向修正：识别学生目标（保研/考研/就业/考公/基层），对匹配路径 +0~8 分（受能力门槛约束，不替代能力评分）
- 兜底压制：考研/考公仅在其他路径均不明显时才可成为主推荐

**可视化与佐证**
- 雷达图：六维能力画像
- 柱状图：九条路径评分对比
- 相似学长学姐案例（Top 5）：基于余弦相似度匹配，隐去姓名/学号/联系方式
- 校企合作单位与岗位推荐：单位、岗位、核心职能、薪资参考、匹配原因
- 联网检索（可选）：配置 Tavily API Key 后可检索外部招聘信息

**AI 辅导员智能问答**
- 多轮对话，注入当前学生画像和路径评分作为上下文
- 6 个常见问题快捷按钮：保研/考研、科研/实习、岗位、三个月规划、谈心谈话、绩点一般竞赛强
- 对话面向辅导员第三人称（"该生……"），内置谈心谈话切入指引
- 无 API Key 时本地规则兜底

**报告导出**
- JSON：完整结构化数据，含画像、评分、算法分析、意向分析、相似案例、岗位推荐
- Word（.docx）：正式评估报告，含雷达图、柱状图、推荐依据、规划建议、案例、岗位、免责声明

### 2.2 管理员端

- **数据导入**：上传历史学生 Excel、校企合作单位 Excel、政策 PDF；支持 LLM 或规则两种结构化方式
- **文件溯源管理**：按来源文件删除导入数据，级联清理
- **规则配置查看**：各路径评分规则与成本优化策略
- **数据看板**：学生总数、单位总数、缓存命中、LLM 状态、路径分布饼图
- **系统设置**：API 配置查看、缓存清理、全量数据重置（危险操作，红色警示）

### 2.3 成本控制

- 政策、学生、单位首次结构化后持久化存储，不重复调用 LLM
- 相似度计算由 scikit-learn 本地完成，不消耗 Token
- 只将 Top-K 匹配结果传给 LLM 生成建议
- 相同输入命中 MD5 缓存，跳过 API 调用
- 无 API Key 时自动降级为启发式抽取 + 规则评分，系统仍可正常运行

## 三、快速开始

3 分钟跑起来：

```bash
# 1. 进入项目目录
cd student_growth_system_commercial

# 2. 创建虚拟环境（推荐）
python -m venv .venv
.venv\Scripts\activate            # Windows PowerShell/CMD
# 或 source .venv/bin/activate    # macOS / Linux

# 3. 安装依赖
pip install -r requirements.txt

# 4. 配置 API Key（可选，不配置时走本地规则模式）
copy .env.example .env            # Windows
# 或 cp .env.example .env         # macOS / Linux
# 编辑 .env 填入 SILICONFLOW_API_KEY

# 5. 启动
streamlit run app.py
```

浏览器自动打开 `http://localhost:8501`，默认进入"用户端｜路径评估"。

### 4.2 依赖说明

主要依赖（见 `requirements.txt`）：

| 库 | 用途 |

| python-docx | Word 报告生成 |

### 4.3 环境变量配置

`.env` 文件放在项目根目录。模板示例：

```env
# ─ 大模型配置（不填则自动走本地规则模式） ───────────────
SILICONFLOW_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
SF_MODEL_NAME=deepseek-ai/DeepSeek-V3.1
SF_BASE_URL=https://api.siliconflow.cn/v1/chat/completions

# ─ 管理员密码（默认 admin123，生产环境务必修改） ───────
ADMIN_PASSWORD=admin123

# ─ 联网检索（可选） ─────────────────────────────────────
TAVILY_API_KEY=
```

> **硅基流动**：国内大模型 API 中转平台，支持 DeepSeek-V3、Qwen、GLM、Llama 等主流模型。注册地址 [https://siliconflow.cn](https://siliconflow.cn)。可选模型见官方文档，常用：
> - `deepseek-ai/DeepSeek-V3.1`（默认，质量高、速度慢）
> - `Qwen/Qwen3-30B-A3B-Instruct-2507`（速度快，质量接近）
> - `Qwen/Qwen2.5-7B-Instruct`（极快，适合抽取类任务）

### 4.4 首次运行验证

启动后进入"用户端｜路径评估"，点击"快速表单"填写一个示例学生，点击"分析学生画像"。若右上角显示**大模型增强模式**（绿色），说明 API 连接正常；显示**本地规则模式**（橙色），则检查 API Key 是否正确或网络是否通畅。

## 五、使用手册

### 5.1 主要页面导航

侧边栏四个入口：

| 入口 | 面向 | 用途 |
|------|------|------|
| 用户端｜路径评估 | 辅导员 | 录入学生信息 → 生成六维画像 → 九条路径评估 → 导出报告 |
| 智能问答｜AI 辅导员 | 辅导员 | 基于当前学生画像，问答谈心谈话策略、帮扶建议 |
| 管理员｜数据导入 | 管理员 | 批量导入历史学生、校企合作单位、政策文件 |
| 管理员｜规则配置 / 数据看板 / 系统设置 | 管理员 | 查看规则、运行统计、API 与缓存管理 |

管理员功能需先点击侧边栏展开区的"管理员登录"输入密码。

### 5.2 场景一：单个学生成长路径评估

**典型工作流**（耗时 1~2 分钟）：

1. 进入"用户端｜路径评估"
2. 选择录入方式：
   - **填写学生自述**：粘贴学生自我介绍或座谈记录
   - **上传简历**：PDF / DOCX / TXT / MD 均可
   - **快速表单**：无简历时手动填写关键字段
3. 选择开关：
   - ☑ 使用大模型抽取（推荐开启，未配置 API Key 时自动回退）
   - ☑ 使用大模型生成最终建议
   - ☐ 联网检索匹配岗位（可选，需 Tavily Key）
4. 点击"分析学生画像"，系统抽取字段并弹出**画像核对**表单
5. 检查绩点、CET4、CET6、志愿时长等关键字段（抽取偶有偏差，人工核对 10~20 秒）
6. 点击"使用微调后的画像生成推荐"→ 出具完整评估结果
7. 查看 4 张指标卡（主推荐路径、最强维度、目标意向、当前模式）+ 雷达图 + 路径柱状图
8. 阅读"综合成长建议"、"相似学长学姐案例"、"校企岗位推荐"
9. 点击"下载 Word 正式评估报告"存档，或"下载 JSON"供后续分析

### 5.3 场景二：谈心谈话准备

1. 先在"用户端"完成该生的画像评估（5.2 流程）
2. 切换到"智能问答｜AI 辅导员"
3. 点击快捷按钮 **"该生情况该如何谈心谈话？"**，或自行输入
4. AI 会结合该生的六维画像、路径评分给出：
   - 切入话题建议
   - 需要关注的学业/心理风险信号
   - 引导方向（如"可提及其竞赛优势，鼓励其补强科研"）
5. 继续追问细化，例如："该生父母期望考公，他本人想就业，如何处理？"

### 5.4 场景三：批量导入历史学生 / 合作单位

1. 展开侧边栏，点击"管理员登录"，输入密码
2. 进入"管理员｜数据导入"
3. 分别上传：
   - 历史学生 Excel（字段参考 §7.1）
   - 校企合作单位 Excel（字段参考 §7.2）
   - 政策 PDF（可选）
4. 选择结构化方式：
   - **LLM 结构化**：高准确度，首次消耗 Token（之后缓存）
   - **规则结构化**：0 Token，字段完整时准确度 80%+
5. 导入完成后，后续所有学生评估都会自动匹配历史相似案例

### 5.5 结果解读指南

**路径得分如何看**
- 主推荐路径：final_score 最高者，通常 >65 分时推荐可信；50~65 分为参考；<50 分提示该路径目前不成熟
- 兜底路径（考研/考公）：若系统压制其分数（显示 ≤42），说明该生有更优路径；若抬升（显示 ≥58），说明其他路径均不明显，建议认真评估兜底选择

**六维画像如何看**
- 雷达图缺角明显的维度 = 短板，可作为帮扶切入点
- 三个月行动计划会针对最大短板给出具体建议

**意向识别**
- 系统从学生目标、自述、经历中识别"保研/考研/就业/考公/基层"意向
- 每种意向最多给对应路径 +8 分，不足以推翻能力评分
- 若未识别到意向，系统会建议辅导员主动询问学生真实方向

### 5.6 常见操作问题

| 问题 | 处理 |
|------|------|
| 抽取结果字段缺失 | 在"画像核对"页面手动补全，再点生成推荐 |
| Word 报告中文乱码 | 检查是否安装中文字体，见 §9.3 |
| 切换学生后上一个学生数据还在 | 正常，`st.session_state` 保留上次结果；重新点"分析学生画像"即可覆盖 |
| 相似案例总是空的 | 管理员未导入历史学生库，先完成 §5.4 |
| LLM 调用超时 | 默认超时 90 秒；网络不稳定时可关闭大模型开关，走本地规则 |
| 缓存占用过大 | 在"系统设置"中点击"清空缓存" |

## 六、命令行工具

除 Streamlit 界面外，`main.py` 提供 CLI 入口，适合批处理和脚本集成。

### 6.1 批量导入历史数据

```bash
python main.py init \
  --students "历史学生信息.xlsx" \
  --companies "校企合作单位.xlsx" \
  --policy   "保研政策.pdf"
```

参数说明：
- `--students`：历史学生 Excel 路径（可省）
- `--companies`：校企合作单位 Excel 路径（可省）
- `--policy`：政策 PDF 路径（可省）

### 6.2 单学生评估（纯本地规则，不调用 LLM）

```bash
python main.py recommend \
  --text "该生绩点2.95，四级480，蓝桥杯省一，美团后端实习4个月，做Java/Redis/MySQL" \
  --no-extract-llm \
  --no-advice-llm
```

### 6.3 输出 JSON / Word 报告

```bash
python main.py recommend \
  --text "该生绩点3.5，有科研和竞赛，希望保研" \
  --output  result.json \
  --output-docx result.docx
```

| 参数 | 说明 |
|------|------|
| `--text` | 学生自述/材料文本（必填） |
| `--file` | 学生简历文件（与 --text 二选一） |
| `--no-extract-llm` | 关闭大模型抽取，走启发式规则 |
| `--no-advice-llm` | 关闭大模型建议生成，走本地规则 |
| `--output` | JSON 报告输出路径 |
| `--output-docx` | Word 报告输出路径 |

### 6.4 批处理脚本示例（PowerShell）

```powershell
# 批量处理某目录下所有简历 PDF，生成 Word 报告
Get-ChildItem .\resumes\*.pdf | ForEach-Object {
    $out = ".\reports\$($_.BaseName).docx"
    python main.py recommend --file $_.FullName --output-docx $out
}
```

## 八、评分与推荐方法

### 8.1 整体流程

```
学生输入文本
    │
    ▼
┌─────────────────────┐
│ 画像抽取            │ ← 大模型 / 启发式规则
└─────────────────────┘
    │
    ▼
┌─────────────────────┐
│ 底层指标评分        │ ← 绩点、英语、竞赛、论文、实习、
│ (12 项 0-100)       │    学生工作、荣誉、志愿、科研等
└─────────────────────┘
    │
    ▼
┌─────────────────────┐
│ 六维能力画像合成    │ ← 加权公式（见 §8.2）
└─────────────────────┘
    │
    ▼                  ▼
┌──────────┐        ┌─────────────┐
│ 规则评分 │        │ TOPSIS 评分 │
│ 9 条路径 │        │ 9 条路径    │
└──────────┘        └─────────────┘
    │                  │
    ▼                  ▼
 final = 0.75 × rule + 0.25 × topsis
    │
    ▼
┌─────────────────────┐
│ 意向修正（+0~8）    │ ← 受能力门槛约束
│ 兜底压制（考研/考公）│
└─────────────────────┘
    │
    ▼
┌─────────────────────┐
│ 相似案例匹配 + LLM 建议生成 │
└─────────────────────┘
```

### 8.2 底层指标与六维合成

**底层指标（12 项，每项 0-100 分）**

| 指标 | 说明 |
|------|------|
| gpa_score | 绩点分。4/5 分制：`60+(gpa-1)×10`；百分制：直接使用 |
| english_score | 英语分。CET-4 ≥425/500/550 → 70/78/82；CET-6 ≥425/500/550 → 85/92/96 |
| honor_score | 荣誉分。国家级 +45、省级 +30、校级 +30，封顶 100 |
| competition_score | 竞赛分。Top3 加权 0.6/0.3/0.1，单项 = 奖项分 × 级别系数 × 贡献度 |
| paper_score | 论文分。Top3 加权 0.7/0.2/0.1，单项 = 基础分 × 作者系数 × 状态系数 |
| research_score | 科研经历分。关键词累加：导师/大创/专利 +45，负责人/一作 +25，CCF/SCI/EI +30 |
| internship_score | 实习分。基础 20，大厂 +45 / 一般企业 +30，技术栈 +25，月数 ×2 封顶 +10 |
| student_work_score | 学生工作分。主席/负责人 +55、班委/社团 +35、荣誉 +25，封顶 100 |
| volunteer_score | 志愿服务分。`min(100, 小时数 × 1.2)`，关键词兜底 40 |
| major_score | 专业方向分。CS/AI/网安 95；电子/数学/自动化 80；其他 65 |
| communication_score | 沟通能力分。启发式关键词扫描 |
| normal_competition_score | 普通推免用的竞赛最高项分（不累加） |

**六维合成公式**

```
学业基础 = 0.70 × gpa + 0.20 × english + 0.10 × honor
科研创新 = 0.65 × paper + 0.35 × research
竞赛实践 = competition（直接使用）
工程实践 = internship（直接使用）
组织服务 = 0.70 × student_work + 0.30 × volunteer
成长规划 = 启发式（基础 15 + 目标 20 + 广度 30 + 跨域 20 + 无短板 15）
```

### 8.3 路径规则评分公式

系统定义 9 条路径，每条路径有差异化权重：

| 路径 | 权重公式（简化） |
|------|---------------|
| 普通推免 | 0.50 绩点 + 0.20 竞赛最高项 + 0.12 论文 + 0.10 科研 + 0.05 英语 + 0.03 学生工作 |
| 特殊专长 A 类 | 0.35 max(竞赛,论文) + 0.20 竞赛 + 0.25 论文 + 0.10 科研 + 0.10 绩点 |
| 特殊专长 B | 0.30 科研 + 0.20 论文 + 0.15 竞赛 + 0.20 绩点 + 0.15 专业 |
| 特殊专长 C / 辅导员 | 0.45 学生工作 + 0.20 志愿 + 0.15 荣誉 + 0.10 沟通 + 0.10 绩点 |
| 支教计划 | 0.35 学生工作 + 0.30 志愿 + 0.15 荣誉 + 0.10 绩点 + 0.10 沟通 |
| 工程专项 / 专硕 | 0.20 绩点 + 0.20 实习 + 0.15 竞赛 + 0.10 论文 + 0.15 科研 + 0.10 专业 |
| 实习就业 | 0.45 实习 + 0.10 专业 + 0.10 竞赛 + 0.05 绩点 + 0.10 科研 |
| 考研 | 0.25 绩点 + 0.15 英语 + 0.15 专业 + 0.15 max(科研,竞赛,实习) + 0.30 基础 70 分 |
| 考公 | 0.25 学生工作 + 0.20 沟通 + 0.15 绩点 + 0.15 荣誉 + 0.10 志愿 + 0.10 专业 + 0.05 英语 |

**基础惩罚项**（乘到前 7 条路径上）：
- 绩点 < 2.8 → × 0.25
- 有挂科 → × 0.3
- CET-4 < 425 → × 0.5

**兜底压制**（apply_fallback_policy）：
若该生有任一"主要路径"分数 ≥ 55 或能力指标达阈值，则考研/考公被压到 ≤ 42；否则抬升到 ≥ 58。

### 8.4 TOPSIS 多指标综合评价

简化版 TOPSIS：
1. 决策矩阵 = 9 条路径 × 6 维画像
2. 标准化、加权
3. 计算正理想解和负理想解
4. 接近度 = `d⁻ / (d⁺ + d⁻)` → 0~100

### 8.5 意向修正

从学生材料中识别 5 类意向（保研/考研/就业/考公/基层），每识别到一类，对相关路径加 0~8 分（受能力门槛约束）：

| 意向 | 能力门槛 | 加分路径 |
|------|----------|----------|
| 保研 | 学业基础 ≥ 60 | 普通推免 +6，特殊专长 A/B +4 |
| 考研 | 学业基础 ≥ 55 | 考研 +6，普通推免 +2 |
| 就业 | 工程实践 ≥ 25 | 实习就业 +8，工程专项 +3 |
| 考公 | 学业基础 ≥ 50 | 考公 +7，支教 +2 |
| 基层 | 组织服务 ≥ 20 | 特殊专长 C +7，支教 +7 |

### 8.6 相似案例匹配

基于余弦相似度 + Jaccard 标签相似度：
```
similarity = 0.70 × cosine(六维) + 0.20 × jaccard(tags) + 0.10 × (同专业 ? 1 : 0)
```
返回 Top 5 历史案例（匿名化）。

### 9.2 Word 正式评估报告

使用 `python-docx` 生成 `.docx` 文件，包含以下 10 个章节：

1. **报告封面** — 标题、生成时间、系统名称、学生姓名
2. **学生基础画像** — 专业、绩点、英语、竞赛、科研、实习、学生工作、目标意向
3. **六维能力画像** — 表格 + 雷达图
4. **路径适配度分析** — 表格 + 柱状图
5. **主推荐路径与备选路径** — 推荐理由、备选、不优先推荐
6. **推荐依据说明** — 评估方法、意向识别、修正、优劣势、风险、建议
7. **未来 1-3 年规划建议** — 近期（3 月）、中期（1 年）、长期（3 年）
8. **相似学长学姐案例** — 相似度、路径、去向、匹配原因（脱敏）
9. **就业/实习/升学建议** — 单位、岗位、核心职能、薪资、匹配原因
10. **免责声明**

### 12.4 安全建议

- **密钥安全**：`.env` 不入 Git，加入 `.gitignore`；生产环境使用环境变量或密钥管理系统
- **身份认证**：管理员页面建议对接学校统一身份认证（CAS / OAuth2）
- **数据隔离**：每个辅导员可独立部署实例，或改造 `storage.py` 按用户 ID 隔离数据目录
- **备份策略**：定期备份 `data/students.json` 和 `data/policy_rules.json`
- **监控**：Streamlit 日志输出到文件，配合 `fail2ban` 或 WAF 防刷
- **学生数据合规**：遵守《个人信息保护法》，报告内自动脱敏；原始简历文件建议定期清理

## 十三、常见问题 FAQ

**Q1: 不配置 API Key 能不能用？**
A：可以。系统会自动降级到"本地规则模式"，使用启发式抽取 + 规则评分 + 本地建议生成。所有功能可用，但建议质量会略低。

**Q2: 抽取的画像字段不准怎么办？**
A：在"画像核对"表单里手动修改任何字段，再点击"使用微调后的画像生成推荐"。系统会基于修改后的数据重新评估。

**Q3: 大模型响应很慢？**
A：DeepSeek-V3.1 单次响应 10~25 秒属正常，一次评估调用 2 次约 30 秒。加速方案：
- `.env` 中将 `SF_MODEL_NAME` 改为 `Qwen/Qwen3-30B-A3B-Instruct-2507`，速度快 3-5 倍
- 关闭"使用大模型生成最终建议"开关，只用本地规则
- 相同学生输入会命中缓存，后续秒级返回

**Q4: 雷达图/柱状图没显示？**
A：检查是否已点击"使用微调后的画像生成推荐"。若仍无，打开系统设置清空缓存后重试。

**Q5: Word 报告中文乱码？**
A：Linux 服务器需安装中文字体，见 §9.3。Windows 一般不会有此问题。

**Q6: 如何修改六维维度？**
A：编辑 `student_growth_system/config.py` 的 `DIMENSIONS` 和 `DIMENSION_LABELS`，并同步修改 `scoring.py:calculate_dimension_scores` 的合成逻辑。

**Q7: 相似案例总是相同的几个？**
A：历史学生库过小时会这样。至少导入 30 条以上历史数据，相似度区分才明显。

**Q8: 每次运行都消耗 Token 吗？**
A：不是。系统使用 MD5 缓存（`data/query_cache.json`）。相同输入只调用一次 LLM，后续从缓存读取，0 Token。

**Q9: 学生隐私如何保护？**
A：
- 姓名/学号/联系方式不入评分上下文，不传给 LLM
- 相似案例默认脱敏（"张\*三"、路径保留、单位保留）
- 上传的简历文件存在本地 `data/uploads/` 下，可在系统设置中清理
- 不会自动把学生数据上传任何第三方服务（LLM 调用只上传抽取后的画像片段）

**Q10: 兼容 GPT-4 / Claude / 国产其他模型吗？**
A：`llm_client.py` 封装的是 OpenAI 兼容格式，改 `SF_BASE_URL` 即可对接任何兼容 OpenAI Chat Completions 协议的模型服务。

**Q11: 可以做成微信小程序/App 吗？**
A：核心评分逻辑全部在 `student_growth_system/` 包内，与 UI 无关。封装 REST API（FastAPI）后，前端可自由替换为小程序、App、企业微信应用等。
